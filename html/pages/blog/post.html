Building Scalable Design Systems with CSS Custom Properties

JD
John Doe
Mar 28, 2026 · 8 min read
Cover Image

Design systems have become the backbone of modern web development. They ensure consistency, speed up development, and reduce maintenance overhead. In this tutorial, we'll explore how to build a scalable design system using CSS custom properties (a.k.a. CSS variables) and the OKLCH color space.

Why CSS Custom Properties?

Custom properties offer several advantages over preprocessor variables like Sass variables:

  • Runtime flexibility — values can be changed dynamically via JavaScript or media queries
  • Cascade-aware — they respect the CSS cascade, enabling contextual overrides
  • No build step — they work natively in all modern browsers
  • Theming — switching between light/dark themes becomes trivial

Setting Up the Token Layer

The foundation of any design system is a well-organized set of tokens. Here's a minimal token structure:

:root {
  /* Colors */
  --primary: oklch(0.55 0.2 260);
  --success: oklch(0.65 0.18 145);
  --warning: oklch(0.75 0.15 85);
  --danger:  oklch(0.6 0.22 25);

  /* Spacing */
  --space-xs: 0.25rem;
  --space-sm: 0.5rem;
  --space-md: 1rem;
  --space-lg: 1.5rem;
  --space-xl: 2rem;

  /* Radius */
  --radius-sm: 0.375rem;
  --radius-md: 0.5rem;
  --radius-lg: 0.75rem;
}

Building Components on Top of Tokens

Once your tokens are defined, components can reference them without hardcoding values. This means changing a single token automatically updates every component that uses it.

"A design system is only as good as its adoption. Make things easy to use and hard to misuse."

Button Component Example

Here's a simple button that leverages tokens for colors, spacing, and border-radius:

.bm-btn {
  padding: var(--space-sm) var(--space-md);
  border-radius: var(--radius-md);
  font-weight: 600;
  transition: all 0.15s ease;
}

.bm-btn-primary {
  background: var(--primary);
  color: white;
}

The OKLCH Advantage

OKLCH (Oklch Lightness Chroma Hue) provides a perceptually uniform color space. Unlike HSL, colors of equal lightness values actually look the same brightness. This is critical for accessible, consistent design systems.

By adjusting only the lightness parameter, you can generate an entire shade scale from a single hue — and every shade will have consistent visual weight.

Theming with CSS Custom Properties

With custom properties, theming is as simple as overriding values in a different scope:

[data-theme="dark"] {
  --bg: oklch(0.15 0.01 260);
  --fg: oklch(0.95 0.01 260);
  --primary: oklch(0.65 0.2 260);
  --card-bg: oklch(0.2 0.01 260);
}

Conclusion

CSS custom properties are the right foundation for modern design systems. Combined with OKLCH colors, they give you the power to build themeable, consistent, and accessible interfaces with minimal overhead. Start small, think in tokens, and let the language do the work.
